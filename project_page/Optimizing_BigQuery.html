🚀 BigQuery Case Study

Data Architecture & Query Cost Optimization

97% Cost Reduction

1. Project Overview

This project focuses on handling 200GB+ blockchain data using BigQuery to improve query efficiency and reduce cost.

2. The Problem

  • High query cost due to full table scans
  • Slow performance with large aggregations
  • Unnecessary data scanning (I/O inefficiency)

3. The Solution

A. Partitioning

Data was partitioned by block_timestamp (daily) to enable partition pruning.

B. Clustering

Data was clustered by from_address to optimize query performance.

4. SQL Implementation

CREATE OR REPLACE TABLE optimized_transactions
PARTITION BY DATE(block_timestamp)
CLUSTER BY from_address
AS
SELECT 
    hash, 
    from_address, 
    value, 
    gas_price, 
    block_timestamp
FROM raw_transactions;

5. Results

Metric Raw Table Optimized Table Gain
Data Processed 29.42 GB 881 MB 97% Saving
I/O Pattern Full Scan Partition Read Efficient
Performance Slow Fast Improved

6. Key Insights

  • Storage design impacts cost more than query complexity
  • Partitioning reduces unnecessary data scans
  • Clustering improves aggregation performance

7. Conclusion

This project demonstrates how proper data architecture in BigQuery can significantly reduce cost and improve performance for large-scale datasets.
